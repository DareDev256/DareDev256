## ü§ù Contribution Guidelines

Interested in contributing to any of my public repos? Here's how:

1. **Fork the repo** and create your branch from `main`
2. **Follow existing conventions** ‚Äî check each repo's `CLAUDE.md` or `FOR_DARE.md` for project-specific rules
3. **Write meaningful commits** ‚Äî use [Conventional Commits](https://www.conventionalcommits.org/) (`feat:`, `fix:`, `docs:`, `test:`)
4. **Test before you PR** ‚Äî if the repo has tests, run them. If it doesn't, consider adding some
5. **Keep PRs focused** ‚Äî one feature or fix per PR. Small, reviewable diffs get merged faster

> **Note:** Some repos (Passion Agent, Viral Clone) are private. If you're interested in collaborating on those, reach out on [LinkedIn](https://www.linkedin.com/in/james-olusoga-00179940/).
